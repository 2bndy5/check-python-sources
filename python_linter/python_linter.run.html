    python_linter.run¶

    Run clang-tidy and clang-format on a list of changed files provided by GitHub’s REST API. If executed from command-line, then [main()][python_linter.run.main] is the entrypoint.

    See also

    • github rest API reference for pulls

    • github rest API reference for repos

    • github rest API reference for issues

    python_linter.run.is_file_in_list(paths: list, file_name: str, prompt: str) → bool[source]¶

    Determine if a file is specified in a list of paths and/or filenames.

    Parameters¶
    paths : list¶

    A list of specified paths to compare with. This list can contain a specified file, but the file’s path must be included as part of the filename.

    file_name : str¶

    The file’s path & name being sought in the paths list.

    prompt : str¶

    A debugging prompt to use when the path is found in the list.

    Returns¶

    • True if file_name is in the paths list.

    • False if file_name is not in the paths list.

    python_linter.run.set_exit_code(override: int | None = None) → int[source]¶

    Set the action’s exit code.

    Parameters¶

    The number to use when overriding the action’s logic.

    Returns:

    The exit code that was used. If the override parameter was not passed, then this int value will describe (like a bool value) if any checks failed.

    python_linter.run.filter_out_non_source_files(ext_list: list, ignored: list, not_ignored: list, lines_changed_only: bool) → bool[source]¶

    Exclude undesired files (specified by user input ‘extensions’). This filter applies to the event’s FILES attribute.

    Parameters¶
    ext_list : list¶

    A list of file extensions that should be attended.

    ignored : list¶

    A list of paths to explicitly ignore.

    not_ignored : list¶

    A list of paths to explicitly not ignore.

    lines_changed_only : bool¶

    A flag that forces focus on only changes in the event’s diff info.

    Returns¶

    True if there are files to check. False will invoke a early exit (in main() when no files to be checked.

    python_linter.run.verify_files_are_present() → None[source]¶

    Download the files if not present.

    Hint

    This function assumes the working directory is the root of the invoking repository. If files are not found, then they are downloaded to the working directory. This may be bad for files with the same name from different folders.

    python_linter.run.list_source_files(ext_list: list, ignored_paths: list, not_ignored: list) → bool[source]¶

    Make a list of source files to be checked. The resulting list is stored in FILES.

    Parameters¶
    ext_list : list¶

    A list of file extensions that should be attended.

    ignored_paths : list¶

    A list of paths to explicitly ignore.

    not_ignored : list¶

    A list of paths to explicitly not ignore.

    Returns¶

    True if there are files to check. False will invoke a early exit (in main()) when no files to be checked.
